Who's it going to be?

What is a Css Selector?

Once you have some HTML markup you need to be able to tie the specific styles with specific elements. You can do this many different ways, I'll show you the most frequently used selectors.

The Class Selector

Use the class attribute in an element to assign the element to a named class. It is up to you what name you choose for the class. Multiple elements in a document can have the same class value.

You would use this type of selector when you have a change that you would want to affect more than one element. I like to think of it in terms of a classroom full of people, if there was a change I wanted to make that would affect them all "change all their shirts to the color red or give them all brown hair. I would use a class selector for this.

HTML Markup with a class attribute

This is an example of HTML markup that has a class attribute. 

    <h1 class='people'> Some Text </h1>

The id Selector

Use the id attribute in an element to assign an ID to the element. It is up to you what name you choose for the ID. The ID name must be unique in the document.

You would use this type of selector if you wanted to make changes to only one HTML element. Think about your drivers license or passport, both of those are unique to you. The same goes for code the id should be unique to only that specific element.

HTML Markup with a id attribute

This is an example of HTML markup that has a id attribute. 

    <h1 id='people'> Some Text </h1>

Stylesheets vs inline styles

The two primary ways of styling an HTML element. The first is inline-styles, and the other is using a stylesheet. Inline-styles are when you put your styles on the HTML element, and a stylesheet is a separate file for your styles. I recommend always putting your styles in a separate file.

Inline Style

    <h1 style='color:blue;font-size:12px;'> Some Text </h1>

Stylesheet

Because the stylesheet it located in a separate file you need to tell the browser where to find it. You do this by including the location in the head of the HTML Document that the stylesheet is associated with. You probably remember the HTML Skeleton from the earlier section. This time we will add a css file to the head section 

     <!DOCTYPE html>
      <html>
        <head>
             <link rel="stylesheet" type="text/css" href="../css/people.css" />
        </head>
        <body>
            <h1 class='people'> Class of People </h1>
            <h1 id='person'> Just one Person </h1>
        </body>
    </html>

Now that the browser knows where to find the file containing your styles you can start to add them 

   .people {
        font-size: 12px;
        color: blue;
    }

    #person {
        font-size: 18px;
        color: red;
    }

How to line it up

What is layout

Many things change the layout of a page. Everything from text alignment to the width of your containers. In the this section I'll review how to change the layout of containers.

Width and Height

When specifying the width and the height of a element you can use several different methods

px and %

The following example we have a div with the class main-container. 

     <!DOCTYPE html>
      <html>
        <head>
             <link rel="stylesheet" type="text/css" href="../css/people.css" />
        </head>
        <body>
            <div class='main-container>  </div>

        </body>
    </html>

We can add some style to adjust the layout 

    .main-container {
        width: 100%;
        height: 400px;
        background-color: grey;
    }

This would render into

If we wanted to do the same thing with a fixed width of 200px this is what the code would look like 

    .main-container {
        width: 200px;
        height: 400px;
        background-color: grey;
    }

This would render into

Two Containers

Two containers would look like this 

     <!DOCTYPE html>
      <html>
        <head>
             <link rel="stylesheet" type="text/css" href="../css/people.css" />
        </head>
        <body>
            <div class='main-container-1>  </div>
            <div class='main-container-2>  </div>

        </body>
    </html>

Add some styles 

    .main-container-1 {
        width: 200px;
        height: 400px;
        background-color: grey;
    }
    .main-container-2 {
        width: 200px;
        height: 400px;
        background-color: blue;
    }

This would render into

Side by Side Container

Two containers side by side would look like this 

     <!DOCTYPE html>
      <html>
        <head>
             <link rel="stylesheet" type="text/css" href="../css/people.css" />
        </head>
        <body>
            <div class='main-container-1-left>  </div>
            <div class='main-container-2-right>  </div>

        </body>
    </html>

Add some styles 

    .main-container-1-left {
        width: 50%;
        height: 400px;
        background-color: grey;
       display: inline-block;
    }
    .main-container-2-right {
        width: 50%;
        height: 400px;
        background-color: blue;
       display: inline-block;
    }

This would render into

Now these containers will line up side by side because I put a display property of inline-block. Which means the element will only take up as much room as needed, which allows for other elements to line up next to it.

Position

Relative? Static? Fixed? Absolute? What?

Sometimes you just want things to stay put

Here are the different positions you can give an element

  • relative : The element is positioned relative to its normal position

  • absolute : The element is positioned relative to its first positioned (not static) ancestor element

  • static : Default value. Elements render in order, as they appear in the document flow

  • fixed : The element is positioned relative to the browser window

  • initial : Sets this property to its default value

  • inherit : Inherits this property from its parent element.
